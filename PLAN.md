# Uliunai.lt Platform — Master Plan

## Overview

A unified platform for the Uliunai.lt Killing Floor community — covering KF1 and KF2 game servers, player statistics, VIP systems, admin panels, and a shared CDN. All under one domain using subdomains.

---

## Monorepo Structure

All platform code lives in this repository (`uliunai-platform`):

```
uliunai-platform/
├── hub/                    # uliunai.lt — main landing site
├── kf1/
│   ├── website/            # kf1.uliunai.lt — KF1 community website
│   ├── statistics/         # kf1.uliunai.lt/statistics — player stats (PHP)
│   └── admin/              # kf1.uliunai.lt/admin — admin panel
├── kf2/
│   ├── website/            # kf2.uliunai.lt — KF2 community website
│   ├── statistics/         # kf2.uliunai.lt/statistics — player stats
│   └── admin/              # kf2.uliunai.lt/admin — admin panel
├── shared/                 # Design tokens, shared configs
├── cdn/                    # Static assets (images, etc.)
├── PLAN.md
└── CLAUDE.md
```

---

## Domain & Subdomain Structure

All subdomains point to one server. One wildcard SSL cert (`*.uliunai.lt` via Let's Encrypt).

| Subdomain | Purpose | Monorepo Path |
|---|---|---|
| `uliunai.lt` | Main landing site — hub to join KF1 or KF2 | `hub/` |
| `kf1.uliunai.lt` | KF1 community website | `kf1/website/` |
| `kf1.uliunai.lt/statistics` | KF1 player statistics (PHP + JS, reads ServerPerksStat.ini) | `kf1/statistics/` |
| `kf1.uliunai.lt/admin` | KF1 admin panel (auth required) | `kf1/admin/` |
| `kf2.uliunai.lt` | KF2 community website | `kf2/website/` |
| `kf2.uliunai.lt/statistics` | KF2 player statistics (same codebase, different .ini) | `kf2/statistics/` |
| `kf2.uliunai.lt/admin` | KF2 admin panel (auth required) | `kf2/admin/` |
| `cdn.uliunai.lt` | Static assets via Cloudflare | `cdn/` |

**Note:** All on ONE domain — no extra cost. Wildcard DNS record `*.uliunai.lt → SERVER_IP`.

## Architecture Decisions

### Data Flow (Statistics)
```
KF Server (round ends)
    → ServerPerks mutator writes ServerPerksStat.ini
    → FTPs file to web server
    → api.php detects file change, rebuilds JSON cache
    → Browsers poll api.php every 60s
    → JS updates DOM in-place (no reload)
```

### Why PHP for Stats?
- KF server FTPs the .ini file — that's hardcoded in the mutator, can't change it
- PHP's `parse_ini_file()` reads it natively in one line
- No build step, no Node runtime needed on the server
- Drop files on any hosting and it works
- File-based JSON caching handles massive traffic

### Why NOT SSE/WebSockets for Stats?
- With thousands of concurrent viewers, SSE holds a PHP process per connection — kills the server
- The .ini file only changes when a round ends (every few minutes)
- Polling once per minute is the correct tradeoff: simple, scalable, good enough freshness

## VIP System — Architecture Notes

From Discord discussion (Greiton, HekuT, Lytair, habib):

- **Payment:** Stripe is the standard. Need to account for Russian players who have limited payment options.
- **Data storage:** MySQL database — NOT in game config files. Separate VIP registry to avoid cluttering uliunaisettings.
- **Game ↔ Website sync:** Two options being considered:
  1. Game server reads VIP status from the website API
  2. Website writes to a MySQL DB that the game server queries
- **Key concern:** Clean architecture — Website, Payment, and Server should be separate concerns, not a mess.
- **HekuT mentioned:** A previous server had a well-structured player info system but hasn't figured out how to replicate it yet.
